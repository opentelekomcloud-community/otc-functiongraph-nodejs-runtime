Bundled third-party components integrated in the Node.js runtime
================================================================

.. list-table:: Bundled third-party components integrated in the Node.js runtime
    :header-rows: 1

    * - Name
      - Usage
      - Version
      - 

    * - q
      - Asynchronous method encapsulation

        .. note::
          Please consider using `JavaScript promises <https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise>`_ instead of Q.
          Native promises are faster, have better tooling support
          and are the future.

          Please consider using JavaScript promises instead of Q.

      - 1.5.1
      - see `q <https://github.com/kriskowal/q>`_ on GitHub

    * - co
      - Asynchronous process control
      - 4.6.0
      - see `co <https://github.com/tj/co>`_ on GitHub

    * - lodash
      - Common tool and method library
      - 4.17.10
      - see `lodash <https://github.com/lodash/lodash>`_ on GitHub

    * - esdk-obs-nodejs
      - OBS SDK for Node.js
      - 2.1.5
      - see `esdk-obs-nodejs <https://github.com/huaweicloud/huaweicloud-sdk-nodejs-obs>`_ on GitHub

    * - express
      - Simplified web-based application development framework
      - 4.16.4
      - see `express <https://github.com/expressjs/express>`_ on GitHub

    * - request
      - Simplifies HTTP invocation and supports HTTPS and redirection.


        .. note::
            The request module has been deprecated and is no longer maintained.
            It is recommended to use alternative libraries for making HTTP
            requests, such as:

          * `Got <https://github.com/sindresorhus/got>`_
          * `axios <https://github.com/axios/axios>`_
          * `node-fetch <https://github.com/node-fetch/node-fetch>`_
          * `superagent <https://github.com/visionmedia/superagent>`_
          * `node-http-proxy <https://github.com/http-party/node-http-proxy>`_
          *  or any other actively maintained HTTP client library that suits your needs.

      - 2.88.0
      - see `request <https://github.com/request/request>`_ on GitHub
